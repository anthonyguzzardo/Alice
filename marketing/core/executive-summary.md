# Alice: Executive Summary

## What It Is

A daily journal that asks one AI-generated question derived from your response history. No streaks. No dashboard. No data surfaced back. A signal engine runs underneath that the user never sees.

## The Position

Only product that learns from user history and never surfaces it back.

```
                SHOWS DATA       HIDES DATA
              ┌──────────────┬──────────────┐
 LEARNS       │ Rosebud      │              │
 FROM USER    │ Reflection   │    ALICE     │
              │ Mindsera     │              │
              ├──────────────┼──────────────┤
 DOESN'T      │ Daylio/WHOOP │ Drift/Halka  │
 LEARN        │ Calm/Oura    │ Presently    │
              └──────────────┴──────────────┘
```

No competitor occupies the upper-right. The moat is philosophical: VC-funded companies measured by engagement will never choose to hide data from users.

## What Makes the Signal Engine Valuable

Signals computed from writing behavior, never shown to user. Signal-to-construct mappings below are hypothesized based on cognitive science literature; longitudinal validation is the research opportunity, not an accomplished fact. See `scientific-foundation.md` for the evidence chain.

| Signal | Hypothesized Construct | Who Cares |
|---|---|---|
| Keystroke intervals (IKI) | Processing speed, motor planning | Pharma, Defense, Clinical |
| Pause architecture | Working memory, lexical retrieval | Clinical, Aging, Insurance |
| Lexical diversity (MATTR) | Vocabulary access over time | All sectors |
| Semantic coherence | Discourse organization, executive function | Enterprise, Defense |
| Hedging density | Epistemic uncertainty, risk aversion | Enterprise, Defense |
| P-burst patterns | Language production fluency | Clinical, Pharma |
| Revision behavior | Error monitoring, cognitive doubt | Clinical, Defense |

Key property: **users can't game metrics they can't see**. This produces data uncontaminated by the metric-gaming and practice effects that degrade explicit cognitive tests.
