## Critical architectural rules (do not violate)

From CLAUDE.md, memory, and discipline established this session:

1. **Prod is signal-store-only.** Embedding (TEI/Qwen) and LLM (Anthropic) work happen LOCALLY only via `npm run dev:full`. Never on prod. (See `project_prod_is_signal_store_only.md`.)
2. **Contamination boundary.** Subject submissions NEVER trigger LLM/embed/signal jobs. The boundary docstrings in `src/pages/api/subject/respond.ts` and `src/pages/api/subject/calibrate.ts` are load-bearing. Calibration boundary lost `runCalibrationExtraction` reference next session.
3. **Every subject gets 30 personal seeds at account creation** (via `seedUpcomingQuestions` called from `create-subject.ts`). Seeds are sacred — never overwritten.
4. **Corpus refresh is ADDITIVE.** `npm run corpus:refresh` + `corpus:approve` appends to `tb_question_corpus`. Never replaces a subject's personal queue. Subjects who haven't burned through their seeds are unaffected.
5. **Logical foreign keys only.** No physical FK constraints. Application code is responsible for cascade deletes. See CLAUDE.md "Cascade Dependencies per Parent Table."
6. **Archival means removal, not stubbing.** When a feature is deprecated, all code goes; data preserved under `zz_archive_*` tables. Applied to alice-negative tonight; calibration extraction next session.
7. **Single source of truth for measurements.** Rust signal engine. No TS fallback. Health endpoint exposes `rustEngine: true/false`.
8. **No em-dashes in user-facing prose.** (Memory: `feedback_no_em_dashes.md`.)
9. **Don't surface responses, future questions, signals, or trait values to the user.** (Multiple memories.)
